- **시작**: "이 슬라이드도 Claude Code로 만들었습니다." (메타 강조)

- **데모 전**: "잠깐 같이 봅시다."
- **데모 중**: 실수해도 당황 없이. "이건 실시간이라 가끔 이렇게 됩니다."
- **마무리**: "오늘 약속한 5가지를 회수하고 끝냅시다."

## 1막 발표 노트

### 1-1 Karpathy 트윗
- 트윗을 영어 그대로 읽고 한국어로 풀어 설명
- "throwaway weekend projects"가 핵심 — 의도가 작았다는 점 강조

### 1-2 타임라인
- 각 항목을 천천히. 청중이 따라올 시간 줘야.
- "2025.12 한 달 만에 20% → 80%" — 이 숫자에서 잠깐 멈춤.

### 1-3 개명
- "Karpathy 본인이 이름을 바꾸자고 했다"는 사실이 신선
- "vibe → agentic" 슬로건처럼 보여주기

### 1-4 Opus 4.7
- 짧게. 1-2분.
- "오늘 데모는 다 이 위에서 돈다"

### 1-5 통계
- 숫자 천천히 읽기. 1.7×, 2.74×, 75%
- "이게 왜 시니어가 게이트키퍼인가"의 데이터 베이스

### 1-6 Replit 사건 ★
- 사건 사실만 담담하게. 과장 없이.
- AI의 자백 부분은 그대로 인용 — 그 자체가 강력
- 이 슬라이드 후 잠깐 침묵

### 1-7 약속
- 5개 점을 짚어가며 약속
- "끝에서 우리가 무엇을 알게 될지" 미리 보여줌

## 2막 발표 노트

### 2-1 ~ 2-4 하니스 정의
- 시니어니까 정의에 까다로움. 정확히 말하기.
- 7개 구성 요소를 손가락 세면서 짚을 정도

- "여러분은 지금 어디에 있나요?" 물어보기

### 2-6 ~ 2-7 컨텍스트

- "이 막대가 1M tokens입니다. 우리가 시작하기 전에 이미 이만큼 차있어요"

### 2-8 데모 ①
- 미리 준비한 프로젝트로 이동
- `/init` → CLAUDE.md 자동 생성 보여주기
- `/context` 명령으로 토큰 사용량
- 간단한 작업 한 번 시켜서 토큰 누적 보여주기

### 2-11 ~ 2-15 서브에이전트 ★★ (정점)
- 단일 vs 멀티 시각화 슬라이드를 충분히 시간 들여 설명
- 데모: 3개 터미널 동시에. 미리 명령어 다 준비해 두기
- 끝나면 `git worktree list` → 머지 → 메인의 깔끔한 히스토리

## 청중 반응별 대응

### "너무 빨라요"

- 핵심 메시지는: "[해당 챕터의 핵심 한 줄]"

- 둘 다 좋고, 워크플로우에 따라 선택.

### "비용이 비싸지 않나?"
